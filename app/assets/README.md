# assets — универсальная графика приложения

Здесь лежит вся графика, которую приложение использует **внутри себя**: логотип, иллюстрации, фоны, декоративные элементы, иконки-файлы и т.п.

## Главное

- Это **часть исходников** приложения. Папка **коммитится в Git** и **публикуется при деплое** (в отличие от `uploads/`, куда попадают файлы, загруженные пользователем).
- Лежит внутри `app/`, потому что на сервер публикуется только `app/`. На сервере путь будет `robtop/assets/...`, в вебе — `apps.tilley.live/robtop/assets/...`.
- В коде ссылаемся **относительным** путём от `index.html`:
  - HTML: `<img src="assets/img/logo.svg" alt="">`
  - CSS: `background-image: url('assets/img/bg.png')`
  - JS: `"assets/img/star.svg"`

## Подпапки

- `img/` — картинки, иллюстрации, фоны (SVG, PNG, WebP, JPG).
- `icons/` — иконки отдельными файлами (если не инлайновый SVG в коде).

Можно добавлять свои подпапки по смыслу (например `img/wishlist/`).

## Рекомендации

- Предпочтительно **SVG** (чёткость на любом экране, маленький размер) для логотипов, иконок и простых иллюстраций.
- **WebP/PNG/JPG** — для фото-графики; перед добавлением сжимать.
- Имена файлов латиницей, в нижнем регистре, через дефис: `cherry-tree.svg`, `home-bg.webp`.

## Чем отличается от `uploads/`

| | `assets/` | `uploads/` |
|---|---|---|
| Что | графика приложения | файлы, загруженные пользователем (фото желаний) |
| В Git | да | нет |
| Деплой | публикуется | не трогается, переживает деплой |
| Путь в БД | не хранится | хранится относительный путь |

# ПЛАН — единое нижнее меню (Bottom HUD) для детей и родителей

**Статус:** спека утверждена (решения Джеффа зафиксированы), готов к поэтапному внедрению.
**Дата:** 2026-06-08 · база v2026.06.08.92.
**Цель:** одно меню — внизу. Верх каждого экрана = только контент приложения / главного. Это освобождает шапку (конец наложениям 🔔/⚙ на действия модуля).

---

## 1. Решения (зафиксированы Джеффом)

1. **Одно нижнее меню (HUD), 5 вкладок, одинаковая структура у ребёнка и родителя:**
   `APPS · BANK · CHAT · NOTIFICATIONS · SETTINGS`.
2. **Меню видно ВСЕГДА**, в том числе внутри открытого приложения. Шапка приложения несёт «назад»; полноэкранные сцены (игра, камера, клавиатура чата) временно прячут меню (`hud({hidden:true})`).
3. **BANK и CHAT убираются из сетки** и **открываются СРАЗУ как приложения в своей вкладке** — без промежуточных «гейт-экранов» (у родителя сейчас `bankSection`/`chatSection` — лендинги с кнопкой «→ в модуль»; их убрать, монтировать сам модуль).
4. **Верхний хром (🔔 колокольчик + ⚙ шестерёнка) убирается отовсюду.** Их функции — во вкладках NOTIFICATIONS и SETTINGS.
5. **NOTIFICATIONS — отдельный экран** (не выезжающая шторка), со списком оповещений + секцией «Настройки» (старт: «Разрешить уведомления на этом телефоне»; место под будущие пункты).
6. **SETTINGS** — экран-вкладка; нижнее меню (HUD) на нём ВСЕГДА видно.

## 2. Маршрутизация вкладок (ядро владеет `nav(tab)`)

| Вкладка | Ребёнок | Родитель |
|---|---|---|
| **APPS** | главный экран — сетка плиток | дашборд — сетка плиток |
| **BANK** | смонтировать модуль `bank` (Копилка) ПРЯМО | смонтировать `bank` с ролью parent (панель Начислить/Штраф внутри модуля) |
| **CHAT** | смонтировать модуль `chat` | смонтировать `chat` |
| **NOTIFICATIONS** | экран оповещений | экран оповещений |
| **SETTINGS** | экран настроек | экран настроек |

- Активная вкладка подсвечена. На NOTIFICATIONS — бейдж непрочитанного.
- Меню строит ОДИН компонент ядра (как сейчас `#kidBar`/`#pdTabs` — body-level, fixed; обходит containing-block-ловушку). Заменяет ОБА существующих бара.

---

## 3. Поведение приложений (мини-апп)

- Открытие из сетки APPS → модуль монтируется, шапка `rt-hdr` (назад → APPS), HUD остаётся (активна APPS).
- **BANK/CHAT — корни вкладок:** открываются сразу, «назад» у них = выход в APPS (или скрыт — полировка). Плитки `bank`/`chat` из сетки удалены.
- «Назад» в шапке = ВНУТРЕННЯЯ навигация модуля (шаги/детали); для верхнего уровня = выход в APPS (дублирует вкладку APPS, но безвреден).
- Полноэкранные сцены (Змейка-игра, камера find, композер chat) прячут HUD через `sdk.ui.hud({hidden:true})` — уже есть.

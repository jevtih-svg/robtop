# ПЛАН — слияние модуля «Задания» в «Копилку», удаление плитки Задания

> Дата: 2026-06-09. Версия приложения: v2026.06.09.02 → **v2026.06.09.03**.
> Заказ Джеффа: «Модуль Задания избыточен. Убрать его полностью, функциональность влить в Копилку: ребёнок подаёт задания, видит лог всех очков и заданий, все подтверждения и оповещения там же».

## 1. Главный вывод анализа

Задания уже сделаны как **ресурс уровня приложения**, а не данные модуля (канон `ГАЙД-задания.md` §1): движок `sdk.tasks` (`app/core/sdk.js`), сервер `api/tasks.php` + `api/_tasks.php`, таблица `tasks`. Модуль `tasks` и блок «Задания» в `bank` это просто **два UI одного источника**. Значит это консолидация интерфейса, а не миграция данных. Сервис остаётся целиком; убираем один из двух фронтендов и доделываем второй.

Решения Джеффа (2026-06-09):
- **Три вкладки** в Копилке: «Задания» (по умолчанию) · «Игры» (лог очков за игры) · «Родители» (лог родительских очков). Лог очков остаётся раздельным как сейчас.
- **Полный паритет** потока: ребёнок логирует сделанное дело со своей оценкой очков → родитель в шторке-ревью правит очки и подтверждает (✓+N) либо отклоняет (✕).

## 2. Что меняется

### `app/modules/bank/` — единственный полный UI заданий
- Три вкладки: «Задания» (деф.) / «Игры» (бывшая «apps») / «Родители» (parents).
- Вкладка «Задания» получает полное секционное управление (порт из модуля tasks):
  - **Родитель:** «Ждут проверки» (предложения детей первыми + проверки выполнения) · «Активные и повторяющиеся» (тап = правка) · «Выполнено» (лог, кап 50) · «+ Новое задание».
  - **Ребёнок:** «Сделать» · «На проверке» · «Выполнено» · «+ Предложить задание».
- Порт `openProposeSheet` (ребёнок) и `openReviewSheet` (родитель, правка очков); `approveTask(tk, pts)` принимает поправку.
- Бейдж-счётчик переезжает с вкладки «Родители» на «Задания».
- Панель родителя (начислить/снять/штраф/бонус), свинка, стрики — без изменений.
- Новые i18n-ключи (секции, propose, review, теги) в en/ru/lv; CSS секций/ревью/предложения в `module.css` (скоуп `.bk-*`).

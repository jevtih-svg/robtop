# RobTop — очки (пункты): движок, политика, подключение модулей

> Канонический документ по системе очков («пункты», копилка). Обновлено: **2026-06-07**.
> Движок: `app/core/sdk.js` (поверхность `sdk.points`). UI: модуль `app/modules/bank/`.
> Прочитай этот файл ЦЕЛИКОМ, прежде чем начислять очки из любого модуля или менять механику.

---

## 1. Устройство (одна минута)

Очки — это **леджер транзакций**: каждая операция добавляет строку «+N / −N с причиной», баланс = сумма всех строк. Ничего не перезаписывается и не удаляется (аддитивность — жёсткий инвариант, как и везде в RobTop).

Где живёт (generic-стор `module_data`, БЕЗ собственных таблиц):

| Коллекция | Что там | Формат `data` |
|---|---|---|
| `bank` / `points` | транзакции | `{n, reason, src, kind, note?}` |
| `bank` / `meta` | винстрик (одна строка) | `{streak}` |
| `bank` / `tasks` | задания от родителей (2026-06-07, §6) | `{title, points, type:"recur"\|"once", timesDone?, lastDoneAt?, claimedAt?, doneAt?}`; статус строки module_data: `active` → `pending` → `active`/`done` |

Писать туда руками НЕЛЬЗЯ — только через `sdk.points.*` (движок сам ведёт винстрик и бонусы). Исключение — `bank/tasks`: это ОБЫЧНЫЕ данные (не леджер), модуль bank правит их через `sdk.data`; но очки за задания всё равно идут только через `sdk.points.add`. Демо-режим (file://, без сервера) работает полностью: тот же движок пишет в localStorage.

⚠️ Если родитель ВЫКЛЮЧИТ модуль «Копилка» в «Управлении приложениями», сервер (`data.php`) начнёт отвечать 403 на записи в `bank/*` — очки из игр ПЕРЕСТАНУТ копиться (тихо, без ошибок у ребёнка). Это осознанное поведение «выключил копилку = выключил очки»; имей в виду при поддержке.

## 2. Поверхность `sdk.points`

```js
sdk.points.add(n, reason, opts?)  // → Promise<{ok, n, kind, streak|null, bonus}> — НИКОГДА не reject
//  n      — целое, со знаком: +10 / −5
//  reason — машинная причина из реестра §4 (своя: "<модуль>_<что>")
//  opts   — {kind?, src?, note?}; kind по умолчанию: win (n≥0) / loss (n<0) / manual (reason *_manual)
//           src по умолчанию — id вызвавшего модуля; note — текст для ребёнка (до 80 зн., НЕ переводится)

sdk.points.get()         // → Promise<number> — баланс (0 при ошибке сети)
sdk.points.summary()     // → Promise<{balance, streak, count, items[]}> — может reject (сеть)
sdk.points.streakReset() // → Promise<bool> — сжечь винстрик (для будущего модуля заданий)
```

## 3. Виды транзакций (kind) и винстрик

| kind | Кто пишет | Винстрик |
|---|---|---|
| `win` / `loss` | игры и приложения | **не трогают** |
| `task_done` | задание родителей выполнено | **+1** (макс **21**) и бонус серии отдельной строкой |
| `task_fail` | задание родителей не выполнено | **сгорает в 0** |
| `daily_bonus` | все задания дня выполнены | не трогает |
| `parent` | родитель произвольно (панель в Копилке) | не трогает |
| `manual` | легаси-поправки (`reason` оканчивается `_manual`) | не трогает |
| `bonus` | ТОЛЬКО сам движок (бонус серии) | не трогает |
| `spend` | траты (Магазин — будущее) | не трогает |

**Винстрик (решение Джеффа 2026-06-07, «дословно по Артёму»):** растёт ТОЛЬКО от заданий родителей (`task_done`), победы в играх его не двигают. Бонус серии = `винстрик − 1` пунктов, начисляется при каждом `task_done` отдельной строкой `streak_bonus` (винстрик 3 → +2; макс винстрик 21 → +20). Невыполненное задание (`task_fail`) сжигает весь винстрик. Винстрик двигают задания Копилки (`bank/tasks`, §6: подтверждение повторяющегося и отметка одноразового) и кнопки родительской панели; автоматическое «закрытие дня» (`streakReset()` при невыполненных + `daily_bonus`) пока не реализовано — кнопка «Все задания дня +5» ручная.

В UI винстрик — «огонёк» в левом верхнем углу Копилки (по эскизу Артёма).

| Источник | `reason` | kind | Пункты | Статус |

| Чистка зубов (полные 2 мин) | `teeth` | win | +10 | 🟢 работает |
| Угадай число — отгадал | `guess_win` | win | +10 | 🟢 работает |
| Угадай число — не отгадал | `guess_wrong` | loss | −5 | 🟢 работает |

| Задание родителей выполнено | `task_done` | task_done | +N задания (родитель задаёт при создании, 1–1000, деф. +10 — решение Джеффа 2026-06-07; кнопка панели — фикс. +10) | 🟢 задания Копилки + панель |

| Бонус серии | `streak_bonus` | bonus | винстрик−1 | 🟢 движок |
| Прогулка с собакой — запись сохранена | `walk_done` | win | +N (настройка родителя в walk/meta.reward: шторка настроек модуля walk; деф. +10, 0 = выкл) | 🟢 работает |
| Змейка — новый рекорд (счёт игры выше прежнего лучшего; ровно одна транзакция в конце игры, проигрыш без рекорда минусов НЕ даёт — решение Джеффа 2026-06-07) | `snake_record` | win | +10 | 🟢 работает |
| Найти предмет (план) | `find_win` | win | +10 | ⚪ модуль в плане |
| Домашний музей — выложил (план) | `museum_add` | win | +10 | ⚪ модуль в плане |
| «…охота» (надиктовка Артёма, нерасшифровано) | — | win | +5 | ❓ уточнить у Артёма |
| Магазин — покупка (план) | `spend` | spend | −цена | ⚪ модуль в плане |

**НЕ дают очков** (решение Артёма): Виш-лист, Настроение дня, Слова наоборот, Смешные имена, Оценка дня. (В надиктовке «Угадай число» попало в оба списка; оставлено КАК ЕСТЬ — даёт очки по утверждённому эскизу guess.)

Новые причины добавляй сюда же (таблица выше — единственный реестр).

Особенность `walk_done` (первый настраиваемый тариф, 2026-06-07): сумма читается модулем walk из его же `walk/meta.reward` в момент сохранения прогулки; начисляется ровно один раз при СОЗДАНИИ записи (дооценка записи «Позже» повторно не платит). Записанная родителем прогулка тоже платит — поэтому манифест bank разрешает edit роли parent (миграция 010), а data.php скоупит запись родителя в леджер ребёнка (семейный пул).

## 5. Как подключить очки к модулю (3 шага)

1. **`module.json`**: добавь `"points"` в `permissions`.
2. **Код**: в момент результата зови движок (fire-and-forget, ошибки глотаются):

```js
if(win) sdk.points.add(10, "find_win");          // kind win — по умолчанию
else    sdk.points.add(-5, "find_giveup");        // kind loss — по умолчанию
```

3. **Подпись в истории Копилки**: доложи ключ `bank.r_<reason>` в СВОИ `messages` (все 3 языка!) — Копилка сама его найдёт:

```js
var MESSAGES={
  en:{ find:{ /* … */ }, bank:{ r_find_win:"Find the Object — found!", r_find_giveup:"Find the Object — gave up" } },
  ru:{ find:{ /* … */ }, bank:{ r_find_win:"Найти предмет — нашёл!", r_find_giveup:"Найти предмет — сдался" } },
  lv:{ find:{ /* … */ }, bank:{ r_find_win:"Atrodi priekšmetu — atrasts!", r_find_giveup:"Atrodi priekšmetu — padevās" } }
};
```

Без ключа Копилка покажет `reason` с пробелами вместо подчёркиваний (некрасиво, но не сломается). `opts.note` подпись перебивает (используется родительской панелью; не переводится — для модулей предпочитай ключи).

Дальше как обычно по ГАЙД-новое-субприложение.md: тариф согласуй с Джеффом, строку — в таблицу §4, проверка в jsdom-смоуке (см. §8).

## 6. Копилка (модуль bank) — что показывает

- Свинка с балансом («пункты: N») + огонёк винстрика (тусклый при 0).
- Вкладка **«Задания»**: kinds `win`, `loss`, `spend` (всё от приложений).
- Вкладка **«Родители»**: СВЕРХУ карточки заданий от родителей (`bank/tasks`), ниже история kinds `task_done`, `task_fail`, `daily_bonus`, `parent`, `manual`, `bonus`. На кнопке вкладки бейдж-счётчик: ребёнку — активные задания, родителю — ждущие проверки.
- **Задания от родителей (2026-06-07, решения Джеффа)**: родитель создаёт задание («+ Новое задание» → название ≤60, очки 1–1000 деф. 10, тип). **Повторяющееся** (`recur`): ребёнок жмёт «Сделал!» → `pending` («ждёт проверки ⏳») → родитель «✓ +N» = `add(N,"task_done",{kind:"task_done",src:"parent",note:название})` — винстрик +1 и бонус серии от движка, задание ВОЗВРАЩАЕТСЯ в `active` (`timesDone`+1); «↩» = вернуть в `active` БЕЗ очков и БЕЗ штрафа (наказание — отдельная кнопка «Не выполнено −10» в панели). **Одноразовое** (`once`): очки ребёнку СРАЗУ при отметке (`src:"bank"`, винстрик тоже +1), статус `done` — выполненное видит только родитель (✕ удаляет). `note`=название задания, история Копилки показывает его вместо generic-подписи. Тап родителя по активной карточке = правка, ✕ = удаление (confirm). Вход родителя — карточка «Копилка» на Обзоре дашборда («⏳ ждут проверки: N» из `bank/tasks` в `parent.php content`) → `RobTop.open("bank")`, прецедент walk: запись скоупится в пул ребёнка (data.php).
- Легаси-строки без `kind` (до 2026-06-07): `teeth_manual` → «Родители», остальные по знаку → «Задания».
- Показывает последние 60 строк вкладки; HUD — «пунктов / винстрик».
- **Панель родителя** (шестерёнка; только роль parent из сессии или демо — PIN упразднён 2026-06-07, кнопка ребёнку не рендерится): «Задание выполнено +10» (двигает огонёк), «Не выполнено −10» (сжигает), «Все задания дня +5», произвольная сумма с причиной. Баланс может уходить в минус — это нормально (как в guess).

## 7. Родительский дашборд и события

- `api/parent.php` отдаёт `points` (сумма леджера) и `streak` (bank/meta) → HUD Обзора и сводная карточка Копилки (`parent.sum.bank`).
- **Начисление с дашборда (v2026.06.07.25):** на Обзоре под HUD кнопка «⭐ Начислить очки» → шторка с чипами +10/+20/+25/+50 и «+…» (своя сумма), обязательное «За что?». Пишет через `RT.createSdk({id:"bank"}).points.add(n,"parent_give",{kind:"parent",src:"parent",note})` — тот же контракт, что у панели в Копилке (§8.3 соблюдён: только движок). Снятие (−N) с дашборда сознательно не делается — для этого панель в Копилке.
- **Скоуп = ВЫБРАННЫЙ на дашборде ребёнок (фикс v2026.06.07.44):** sdk.js шлёт `child=<id>` с каждым запросом роли parent (данные, очки, события), data.php проверяет права (`rt_can_manage_child`) и работает со скоупом именно этого ребёнка; у каждого ребёнка СВОЯ копилка. До фикса сервер молча брал ПЕРВОГО ребёнка семьи — при нескольких детях родитель видел одну копилку «на всех» и начислял не тому. Без `child` (старые клиенты из кэша) — прежний фолбэк «первый ребёнок». Модули с `"familyPool":true` в манифесте (walk) — исключение: пул один на семью, селектор ребёнка на них не влияет.
- Модуль `bank` скрыт из ленты Журнала (шум: каждая транзакция зеркалится generic-событием `created`). Доменная история и так видна в самой Копилке; отдельные `events.track` движок НЕ пишет сознательно (не плодим третью запись на транзакцию).
- Источник правды — всегда леджер `bank/points`, никаких кэшей баланса нигде нет (баланс всегда пересчитывается из строк; винстрик — единственное состояние, в `bank/meta`).

## 8. Правила безопасности механики (нарушение = баг)

1. Транзакции НЕ редактируются и НЕ удаляются. Ошибся — компенсируй новой транзакцией (родительская панель, «произвольно»).
2. Не выдумывай новые `kind` — только §3; новый kind = правка движка + этого гайда + Копилки (вкладки).
3. Не пиши в `bank/points`/`bank/meta` через `sdk.data` из других модулей — только `sdk.points.*`.
4. Тарифы — только из §4; новые согласуются с Джеффом и вписываются в таблицу.
5. В jsdom-смоуке модуля с очками проверяй: победа пишет ровно одну строку с верными `n/reason/src/kind`; демо-режим пишет в localStorage (`robtop_md_bank_points`).
6. `sdk.points.add` никогда не кидает — НЕ оборачивай в свою обработку ошибок с ретраями (двойное начисление хуже потерянного).

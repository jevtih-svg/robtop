# RobTop — настройка и релиз

Цель: **один раз** настроить (подробно ниже), а дальше выпускать обновления **в одно действие** — просто отправить изменения в ветку `main`, и сайт сам обновится на Hostinger.

```
  ты вносишь правки
        │
        ▼
   git push в main ──────────►  GitHub Actions  ──FTPS──►  Hostinger
   (или кнопка Run workflow)      (авто-сборка)            apps.tilley.live/robtop
                                                                │
                                              PHP API  ◄──►  MySQL (статистика)
```

Как это работает: на сервер публикуется содержимое папки `app/`. Клиент (`index.html`) общается с PHP-API (`api/*.php`), а тот пишет всё в MySQL. Пароли БД лежат в `config.php` **только на сервере** и в Git не попадают.

## Часть 1. Разовая настройка (делается один раз)

### Шаг 1. Репозиторий на GitHub

1. Создай новый репозиторий на github.com (например `robtop`), приватный.
2. На своём компьютере открой папку проекта (`Приложение RobTop`) в терминале и выполни:

```bash
git init
git add .
git commit -m "RobTop: первый релиз"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/robtop.git
git push -u origin main
```

> `config.php` не закоммитится — он в `.gitignore`. Это правильно.

### Шаг 2. База данных на Hostinger

1. hPanel → **Базы данных → Управление базами данных MySQL**.
2. Создай новую базу и пользователя (пароль сохрани). Запомни:
   - имя базы (вид `u1234567_robtop`),
   - имя пользователя (вид `u1234567_robtop`),
   - пароль,
   - хост (обычно `localhost`).
3. Открой **phpMyAdmin** (кнопка рядом с базой) → вкладка **Импорт** → выбери файл `schema.sql` из проекта → **Вперёд**. Появятся таблицы `users`, `wishlist_items`, `events`.

### Шаг 3. Папка сайта и `config.php` на сервере

1. hPanel → **Файлы → Файловый менеджер**.
2. Зайди в корень сайта `apps.tilley.live` (обычно `.../public_html`) и создай папку **`robtop`** (если её ещё нет).
3. Внутри `robtop` создай файл **`config.php`** со своими данными базы (за образец — `config.example.php`):

```php
<?php
return [
    'db_host'    => 'localhost',
    'db_name'    => 'u1234567_robtop',
    'db_user'    => 'u1234567_robtop',
    'db_pass'    => 'ТВОЙ_ПАРОЛЬ_БД',
    'db_charset' => 'utf8mb4',
    'api_token'  => '',
];
```

> Этот файл деплой не трогает и не перезаписывает. Пароли остаются только здесь.

4. hPanel → **PHP Configuration** → выбери **PHP 8.x**.

### Шаг 4. Данные FTP

1. hPanel → **Файлы → FTP-аккаунты**. Возьми:
   - **FTP host** (вид `ftp.tilley.live` или IP),
   - **FTP username**,
   - **FTP password** (можно задать/сбросить там же).
2. Узнай **полный путь к папке `robtop`**: в Файловом менеджере зайди в `robtop`, скопируй путь из адресной строки. Он понадобится с `/` в конце, например:
   `/domains/apps.tilley.live/public_html/robtop/`
   (у разных аккаунтов путь отличается — бери свой).

### Шаг 5. Секреты в GitHub

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**. Добавь 4 секрета:

| Имя секрета       | Значение                                              |
|-------------------|-------------------------------------------------------|
| `FTP_SERVER`      | FTP host из шага 4 (без `ftp://`)                     |
| `FTP_USERNAME`    | FTP-логин                                              |
| `FTP_PASSWORD`    | FTP-пароль                                             |
| `FTP_REMOTE_DIR`  | путь к папке `robtop/` со слешем в конце (шаг 4)       |

### Шаг 6. Первый деплой

- Либо сделай любой коммит в `main` (`git push`),
- либо в репозитории: вкладка **Actions → Deploy RobTop to Hostinger → Run workflow**.

Открой `https://apps.tilley.live/robtop/` — приложение должно работать с базой. Готово.

## Часть 2. Как выпускать обновления (каждый раз)

Когда я (или ты) меняем файлы в папке проекта:

```bash
git add .
git commit -m "что изменили"
git push
```

Через ~1 минуту изменения уже на сайте. Прогресс виден во вкладке **Actions**.

**Кнопка вместо авто-деплоя.** Если хочешь деплоить вручную: вкладка **Actions → Deploy RobTop → Run workflow**. Это та самая «кнопка в один клик».

**Откат.** Нужно вернуть прошлую версию — на GitHub: **Actions** → открой удачный прошлый запуск → **Re-run jobs**. Либо `git revert <коммит>` и `git push`.

## Альтернатива: деплой кнопкой прямо в hPanel (без секретов)

Если не хочешь возиться с FTP-секретами, у Hostinger есть встроенный Git:

1. hPanel → **Advanced → GIT**.
2. **Create repository**: вставь URL репозитория, ветку `main`, путь установки `public_html/robtop`.
3. Появится кнопка **Deploy** — нажал, и свежая версия с GitHub раскатана. Можно включить **Auto-deployment** (webhook в GitHub), тогда деплой по push.

> Для приватного репозитория добавь SSH-ключ Hostinger в GitHub (Deploy keys). Для публичного — ничего не нужно. В этом варианте `config.php` так же создаётся на сервере и в Git не хранится.

Можно использовать любой из двух способов. GitHub Actions удобнее для «push = в проде», hPanel Git — для «нажал кнопку Deploy».

## Проверка и частые вопросы

- **Открывается, но без данных / ошибка БД** → проверь `config.php` (логин/пароль/имя базы), что `schema.sql` импортирован, и что выбран PHP 8.x.
- **Actions падает на FTP** → проверь `FTP_SERVER/USERNAME/PASSWORD` и что `FTP_REMOTE_DIR` оканчивается на `/` и существует.
- **Фото не сохраняется** → большие картинки. Клиент сам уменьшает их до ~900px, но при необходимости в hPanel → PHP Configuration можно поднять `post_max_size`/`upload_max_filesize`.
- **Сайт открыли локально (файл с компьютера)** → это демо-режим (данные в браузере). Боевые данные — только на сервере с базой.

## Безопасность

- `config.php` не в Git и не публикуется (`.gitignore` + правило в `.htaccess`).
- Внутренние файлы API (`_db.php`, `_bootstrap.php`) закрыты от прямого доступа `.htaccess`.
- При желании можно включить простой токен API: задать `api_token` в `config.php` (тогда добавим заголовок в клиент).

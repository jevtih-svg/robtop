## 1. Поверхности интерфейса — что ЕСТЬ сейчас

### 1.1 Lock-экран (вход в приложение, `index.html` + `shell.js`)
- Форма входа: email (родитель) или никнейм (ребёнок) + пароль.
- Вход по одноразовому 1234 → обязательная смена пароля прямо здесь.
- Кнопка «Я новый родитель — создать семью» → регистрация (никнейм, email, пароль) → вход.
- До подтверждения сессии плитки не рендерятся и реестр модулей не запрашивается.
- Демо-режим (открытие file://) работает без входа, с бейджем demo.

### 1.2 Главный экран
- РЕБЁНОК и демо: активные плитки всегда сверху, ниже за разделителем «Скоро будут» — плитки «скоро»; кнопка ⚙, индикатор версии слева внизу (`#appver`), фон-анимация. HUD-счётчики «приложений/доступно» с v2026.06.07.10 убраны — HUD виден только внутри модулей.
- РОДИТЕЛЬ (с v2026.06.07.6): вместо плиток — **родительский дашборд** (`core/parent.js`, секция `#parent`), read-only: вкладки Обзор (период 7/30, HUD событий/активных дней/очков, график по дням, разбивка и сводка по приложениям, заметки-инсайты) / Виш-лист ребёнка (разделы, деталь с историей, фото по правилу приватности) / Журнал (фильтры по модулям, шум `opened_*`/`viewed_*` и `bank` скрыт). Свой таббар внизу (глобальный HUD скрыт), ⚙ и переключение детей в топбаре. Возврат из любого модуля (`showHome`) ведёт родителя на дашборд. Данные: `api/parent.php`.

### 1.3 Настройки (⚙), по ролям
С v2026.06.07.10 настройки — **отдельный экран** (`#settings`, не шторка): «назад» слева вверху (возврат: ребёнок → плитки, родитель → дашборд), язык — сегмент-контрол в одну строку, «Выйти» — одна кнопка внизу экрана.
| Секция | Родитель | Ребёнок | Демо |
|---|---|---|---|
| Аккаунт (кто вошёл / форма входа); «Выйти» вынесен вниз экрана | ✅ | ✅ | подсказка |
| Семья: список детей, сброс пароля → 1234, блокировка/разблокировка, ＋ добавить ребёнка, ✉ пригласить второго родителя (ссылка + копировать) | ✅ | — | — |

| Управление приложениями | ✅ без PIN | — | ✅ (PIN 1234) |

### 1.4 Управление приложениями («магазин», админка приложения)
- Список всех модулей: вкл/выкл, порядок (▲▼), удаление установленных, установка из `.json`-бандла.
- Родитель попадает без PIN (гейт `rt_admin_gate`: родительская сессия). PIN-форма остаётся только как fallback для не-родителя/демо.
- Отдельной «десктопной админки» нет и не планировалась: это «Управление приложениями» в настройках (шторка магазина открывается поверх экрана настроек), на десктопе и мобильном одинаково.

### 1.4б admin.html (мастер-админка, с 2026-06-07)
- Отдельная консоль `app/admin.html` + `api/admin.php`, гейт `rt_require_admin` (родитель + флаг в `admins`, миграция 008). Спецификация: `ПЛАН-мастер-админка.md`.
- Разделы: дашборд-статистика, пользователи (переименовать/сброс→1234/блок/удалить/права админа/очки), семьи (привязать-отвязать родителя, перенос ребёнка, полный БАН с запретом повторной регистрации по email-хэшу), приглашения, контент модулей (generic-CRUD, картинки только фактом), очки (балансы+транзакции), письма (триггеры+произвольное), приложения (вкл/выкл/порядок глобально), журнал, админы и баны.
- Первый админ — одноразовый SQL (см. план §1), дальше гранты из панели.

### 1.5 family.html (отдельная консоль семьи, переходная)
- Вход/регистрация (вкладки), смена 1234.
- Дашборд: члены семьи, дети; сброс пароля ребёнку; «Передать родителю» (transfer_child по email).
- История приглашений: статус, отозвать, переслать.
- Посадочные страницы ссылок: `family.html?invite=<код>` (принятие приглашений всех типов) и `family.html?reset=<код>` (ввод нового пароля).
- Наружу уходят КОРОТКИЕ ссылки (v2026.06.07.5): `…/robtop/i/<КОД>` (приглашение, код 6 знаков) и `…/robtop/r/<КОД>` (сброс, 8 знаков) — корневой `.htaccess` 302-редиректит их на посадочные. Алфавит без похожих символов (0/O/1/I/L нет), регистр на входе нормализуется, в БД только sha256-хэш кода, старые длинные токены тоже принимаются.

## 3. Чего НЕТ в интерфейсе приложения (разрывы)

Отсортировано по важности.

1. **«Забыли пароль?» на lock-экране** — API есть (`forgot`), UI только в family.html. Родитель, забывший пароль, из приложения выхода не найдёт.
2. **Передача ребёнка настоящему родителю** — ключевой сценарий провизорного опекунства (§4 плана). API + письмо готовы, в приложении кнопки нет (только family.html).
3. **Список отправленных приглашений + отозвать/переслать** — в приложении не виден статус (ожидает/принято/истекло). API готов.
4. **Правила безопасности для ребёнка при первом входе** — в прототипе есть обязательный экран (НЕ фотографируй людей, НЕ загружай свои фото, только простые предметы/картинки из интернета, «если просят другое — скажи родителю»), в приложении его НЕТ. Для детского приложения это пробел №1 по духу проекта.
5. **Принятие приглашения внутри приложения** — `index.html` не понимает `?invite=`/`?reset=`; посадочная — family.html (ссылки с v2026.06.07.3 строятся прямо на неё, так что путь рабочий, но стилистически family.html ≠ приложение).
6. **Добровольная смена пароля** — сменить пароль без сброса нельзя нигде (есть только форс-смена 1234 и сброс родителем/по почте).
7. **Временные опекуны в настройках** — родитель видит только детей; список «дети друзей под моим присмотром» с пометкой «временно» не выводится (API members/children отдаёт тип опекунства).
8. **Приватность фото виш-листа НЕ enforced** — правило «фото видят только ребёнок и его ПРЯМОЙ родитель, временный — никогда» (§4.7/§8 плана) задокументировано, но `uploads/` раздаётся веб-сервером как статика без проверки прав. Нужен отдающий PHP-эндпоинт с `rt_can_view_images()` + deny на прямой доступ к папке.
9. Мелочи: у приглашений в приложении не показывается срок жизни (7 дней — теперь в подсказке «Друзья», у co_parent нет); нет экрана «аккаунт заблокирован» (ребёнок получит общую ошибку входа).

## 4. Серверные шаги-условия (не код)
1. В `config.php` на сервере проверить/поправить `app_base_url` → `https://apps.tilley.live/robtop/app/` (пример обновлён). С v2026.06.07.5 короткие ссылки строятся корректно при обоих значениях (хвост `app/` срезается), но единообразие лучше.
2. После проверки обоих входов: `'single_user' => false` (закрыть анонимный API).
3. Когда понадобятся реальные письма: `mail_driver='brevo'` + ключ + DKIM (инструкция в РЕЛИЗ.md), тест через `mail_test.php`.

## 5. Решено и работает (для полноты)
Музыка при чистке зубов; индикатор версии + cache-busting; полный бэкенд аккаунтов (семьи, опекунства primary/provisional, приглашения 4 типов, сессии, логи «детской системы»); lock-экран; семья и друзья в настройках; блокировка с убийством сессий; письма по трёхъязычным шаблонам; деплой вебхуком с раскладкой `/robtop/app/`; документация закрыта `.htaccess`.
